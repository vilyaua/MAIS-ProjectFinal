# User Story

> Build a CLI password generator that supports custom length, uppercase, lowercase, digits, and special characters with a strength meter

## Specification: CLI Password Generator with Strength Meter

**Complexity:** medium

### Requirements

1. The CLI tool must allow users to specify the desired password length as an input parameter.
2. The tool must allow users to include or exclude uppercase letters, lowercase letters, digits, and special characters in the password.
3. The password generator must validate user inputs for length (e.g., positive integers within a reasonable range) and character type selections (at least one character type must be selected).
4. The tool must generate passwords that comply with the selected character types and length.
5. The tool must implement a password strength meter that evaluates the generated password's strength based on length, character variety, and randomness.
6. The tool must provide clear error messages for invalid inputs.
7. The CLI must output the generated password and the corresponding strength indication to the user.

### Acceptance Criteria

1. Given valid inputs for length and character type selections, when the password is generated, then the output password matches the specified criteria.
2. Given a length input that is not a positive integer or out of allowed range, when the user inputs, then the tool returns an appropriate error message.
3. Given no character types are selected, when the user tries to generate a password, then the tool returns an error indicating at least one character type must be chosen.
4. Given a generated password, when the strength meter evaluates it, then it correctly reflects strength categories (e.g., weak, moderate, strong) based on defined rules.
5. Given a set of inputs, when the password is generated multiple times, then each password meets the criteria and is unique/random.
6. Given the CLI execution, when completed successfully, then the password and its strength score are displayed to the user.
