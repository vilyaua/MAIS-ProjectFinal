# User Story

> Write a Python CLI tool that reads a CSV file and generates a statistical summary report: column types, min/max/mean/median, missing values, and top-5 most frequent values per column

## Specification: CSV Statistical Summary CLI Tool

**Complexity:** medium

### Requirements

1. The tool must accept a CSV file path as input from the command line.
2. It should read and parse the CSV file correctly, handling common CSV formats.
3. The tool should identify column data types (e.g., numeric, categorical, datetime, text).
4. For numeric columns, it must calculate and report min, max, mean, and median values.
5. For all columns, it should report the count of missing values.
6. For categorical/text columns, it must identify and report the top-5 most frequent values with their counts.
7. The tool should handle input validation, such as verifying file existence and correct CSV format, and provide meaningful error messages for invalid input or parsing errors.

### Acceptance Criteria

1. Given a valid CSV file with various data types, when the CLI tool is executed, then it outputs a summary report containing column types and statistics as specified.
2. Given a CSV file with missing values, when the tool is run, then the report accurately reflects the count of missing values per column.
3. Given a numeric column, when the tool processes it, then it correctly calculates and displays min, max, mean, and median values.
4. Given a categorical column, when the tool processes it, then it lists the top-5 most frequent values with their frequencies.
5. Given an invalid file path or non-CSV input, when the tool is run, then it displays a clear error message and exits gracefully.
6. Given an empty CSV file or a CSV with no rows, when the tool is run, then it handles this edge case without crashing and informs the user appropriately.
