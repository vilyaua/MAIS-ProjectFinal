# User Story

> Build a REST API with Flask for a task manager with SQLite persistence, user authentication via API keys, and CRUD operations for tasks with priorities and due dates

## Specification: Task Manager REST API with Flask and SQLite

**Complexity:** medium

### Requirements

1. Implement a Flask REST API to manage tasks with SQLite database persistence.
2. Support user authentication via API keys for all endpoints.
3. Enable CRUD (Create, Read, Update, Delete) operations for tasks.
4. Each task must include fields: title, description, priority (e.g., low, medium, high), and due date.
5. Validate all input fields, ensuring required fields are provided and data formats (e.g., dates) are correct.
6. Handle error cases gracefully with appropriate HTTP status codes and error messages.
7. Ensure API endpoints are protected and accessible only by authenticated users.

### Acceptance Criteria

1. Given a valid API key, when creating a task with all required fields, then the task is saved and a 201 Created response is returned.
2. Given an invalid or missing API key, when accessing any endpoint, then a 401 Unauthorized response is returned.
3. Given a task ID and a valid API key, when retrieving the task, then the task details are returned with a 200 OK response.
4. Given a valid API key and task ID, when updating a task with valid data, then the task is updated and a 200 OK response is returned.
5. Given a valid API key and task ID, when deleting the task, then the task is removed and a 204 No Content response is returned.
6. Given invalid input data (e.g., missing title or invalid date format) when creating or updating a task, then a 400 Bad Request response with error details is returned.
7. Given a request to list all tasks with a valid API key, then a list of tasks with all details is returned with a 200 OK response.
