# User Story

> Create a Python library that converts Markdown text to HTML, supporting headers, bold, italic, links, code blocks, and lists

## Specification: Markdown to HTML Converter Library

**Complexity:** medium

### Requirements

1. The library must provide a function to convert input Markdown text to HTML output.
2. The converter must support the following Markdown features: headers (levels 1-6), bold text, italic text, hyperlinks, code blocks, and unordered/ordered lists.
3. The library must validate the input to ensure it is a string and handle empty or null inputs gracefully.
4. The output HTML must be well-formed, with properly nested and closed tags.
5. The library should handle nested formatting correctly, such as bold text within headers or italic within lists.
6. The library must handle edge cases such as incomplete or malformed Markdown syntax without crashing, providing reasonable fallback HTML or error messages.
7. The library should offer clear error handling for unsupported Markdown features or invalid input types.

### Acceptance Criteria

1. Given valid Markdown input containing headers (# to ######), when converted, then the output includes corresponding <h1> to <h6> HTML tags.
2. Given Markdown text with bold syntax (**text**), when converted, then the output wraps the text in <strong> tags.
3. Given Markdown text with italic syntax (*text*), when converted, then the output wraps the text in <em> tags.
4. Given Markdown with links [text](url), when converted, then the output includes <a href="url">text</a>.
5. Given Markdown code blocks delimited by triple backticks (```), when converted, then the output wraps code content inside <pre><code> tags.
6. Given Markdown lists (unordered with *, ordered with numbers), when converted, then the output correctly produces <ul> or <ol> elements with nested <li> items.
7. Given empty or non-string input, when converted, then the function returns an empty string or raises an appropriate error without crashing.
