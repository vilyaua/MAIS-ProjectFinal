# User Story

> Create a Python library that converts Markdown text to HTML, supporting headers, bold, italic, links, code blocks, and lists

## Specification: Markdown to HTML Converter Library

**Complexity:** medium

### Requirements

1. Support conversion of Markdown headers (e.g., #, ##, ###) to corresponding HTML header tags (<h1>, <h2>, <h3>, etc.).
2. Support conversion of bold text (wrapped in ** or __) to <strong> tags in HTML.
3. Support conversion of italic text (wrapped in * or _) to <em> tags in HTML.
4. Support conversion of Markdown links ([text](url)) to HTML anchor tags (<a href="url">text</a>).
5. Support conversion of code blocks delimited by triple backticks (```) to <pre><code> HTML blocks preserving content formatting.
6. Support conversion of unordered lists marked by -, *, or + to <ul><li> HTML structures and ordered lists marked by numbers to <ol><li>.
7. Validate Markdown input is a string; handle empty and malformed Markdown gracefully with appropriate error handling or fallback rendering.

### Acceptance Criteria

1. Given a Markdown string with # header, when converted, then output HTML contains corresponding <h1> tag.
2. Given Markdown text with **bold** and *italic* syntax, when converted, then output HTML contains <strong> and <em> tags respectively.
3. Given a Markdown link [text](https://example.com), when converted, then output HTML contains an anchor tag <a href="https://example.com">text</a>.
4. Given a Markdown code block with triple backticks, when converted, then output HTML contains a <pre><code> block preserving content.
5. Given a Markdown list using - and 1. syntax, when converted, then output HTML contains <ul> and <ol> tags with nested <li> elements.
6. Given empty Markdown input, when converted, then output is an empty HTML string.
7. Given malformed Markdown input, when converted, then no unhandled exceptions occur and output is reasonable HTML or an error message.
