## Specification: ASCII Art Generator from Text

**Complexity:** simple

### Requirements

1. The script must accept text input from the user via command line or function argument.
2. The script should convert the input text into stylized ASCII art representation.
3. Support for at least one ASCII art font style (e.g., standard block letters).
4. Provide input validation to ensure non-empty text input and handle invalid characters gracefully.
5. Include error handling for unexpected input types or empty inputs.
6. Display the ASCII art output in the console or return as a string for further usage.
7. Handle edge cases such as very long text inputs or special characters by either truncating, sanitizing, or providing meaningful error messages.

### Acceptance Criteria

1. Given a valid text input, when the script runs, then it outputs readable ASCII art corresponding to the input text.
2. Given an empty input or whitespace only, when the script runs, then it raises or returns a meaningful error message indicating invalid input.
3. Given special or unsupported characters in the input, when the script runs, then it either ignores them with a warning or replaces them with a placeholder in the ASCII art output.
4. Given a very long text input, when the script runs, then it either processes the input correctly or provides a notice about input length limitations.
5. Given valid input, when the ASCII art is generated, then the output is printed to the console or returned as a string without runtime errors.
