# User Story

> Build a REST API with Flask for a task manager with SQLite persistence, user authentication via API keys, and CRUD operations for tasks with priorities and due dates

## Specification: Task Manager REST API with Flask and SQLite

**Complexity:** medium

### Requirements

1. Design and implement a Flask REST API for managing tasks.
2. Use SQLite as the database to persist task and user data.
3. Implement user authentication using API keys. Only authenticated users can access the API endpoints.
4. Support CRUD (Create, Read, Update, Delete) operations for tasks.
5. Each task must have attributes: title, description, priority (e.g., low, medium, high), and due date.
6. Validate all inputs, ensuring required fields are present and data types/formats are correct (e.g., date format, priority value).
7. Handle error cases such as invalid API keys, missing or invalid input data, and database errors gracefully with appropriate HTTP status codes and messages.

### Acceptance Criteria

1. Given a valid API key, when a user creates a task with all required fields, then the task is saved to the database and a success response is returned.
2. Given an invalid API key, when a request is made to any endpoint, then the API returns an unauthorized error.
3. Given a valid API key, when fetching tasks, then a list of the user's tasks is returned in JSON format.
4. Given a valid API key, when updating a task with valid changes, then the updated task is saved and the updated data is returned.
5. Given a valid API key, when deleting a task by ID, then the task is removed from the database and a success response is returned.
6. Given invalid input data (e.g., missing required fields or invalid priority), when attempting to create or update a task, then the API returns a validation error with appropriate messages.
