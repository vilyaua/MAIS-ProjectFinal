# User Story

> Build a REST API with Flask that manages a book collection - CRUD endpoints, search by title/author, and input validation

## Specification: Book Collection Management REST API

**Complexity:** medium

### Requirements

1. Implement CRUD endpoints for books: create, read, update, and delete book records.
2. Each book record must include attributes: title, author, publication year, and ISBN.
3. Implement search functionality allowing filtering books by title and author with partial case-insensitive matching.
4. Validate input data on create and update endpoints: title and author required non-empty strings, year must be a valid integer, ISBN must follow a valid format.
5. Handle error cases gracefully with appropriate HTTP status codes and error messages, e.g. 400 for invalid input, 404 for not found.
6. Ensure the API responses are JSON formatted consistently.
7. Support edge cases such as no search results, updating non-existent books, and deleting already deleted books.

### Acceptance Criteria

1. Given valid book data, when creating a book via POST /books, then the API responds with 201 status and the created book data.
2. Given a book ID, when retrieving the book via GET /books/{id}, then the API returns 200 with correct book details or 404 if not found.
3. Given partial title or author parameters, when searching via GET /books?title=abc&author=xyz, then the API returns 200 with all matching books.
4. Given invalid input data, when creating or updating a book, then the API returns 400 with descriptive error messages.
5. Given a book ID, when updating or deleting a book, then the API returns 200 for success or 404 if the book does not exist.
6. Given a search query with no matches, when calling the search endpoint, then the API returns 200 with an empty list.
7. Given inconsistent or malformed JSON input, when calling create or update endpoints, then the API returns 400 with an error message.
