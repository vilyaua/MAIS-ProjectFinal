# User Story

> Write a Python CLI tool that reads a CSV file and generates a statistical summary report: column types, min/max/mean/median, missing values, and top-5 most frequent values per column

## Specification: CSV Statistical Summary CLI Tool

**Complexity:** medium

### Requirements

1. The tool must accept a CSV file path as input via command-line arguments.
2. The tool must read the CSV file and identify the data type of each column (e.g., numeric, categorical).
3. The tool must calculate for each column: minimum, maximum, mean, median values where applicable.
4. The tool must report the number of missing values per column.
5. The tool must determine and display the top-5 most frequent values for each column.
6. The tool must handle input validation, including file existence, file format correctness, and readable CSV structure.
7. The tool must produce a well-formatted textual summary report suitable for CLI output.

### Acceptance Criteria

1. Given a valid CSV file with numeric and categorical columns, when the tool is run, then it outputs a summary with correct column types and statistics.
2. Given a CSV file with missing values, when the tool is run, then the report includes the correct count of missing values per column.
3. Given a column with numeric data, when the tool is run, then min, max, mean, and median values are correctly calculated and displayed.
4. Given a column with categorical data, when the tool is run, then the top-5 most frequent values with their counts are correctly reported.
5. Given an invalid or missing CSV file path, when the tool is run, then it shows an appropriate error message and exits.
6. Given a CSV file where some columns contain mixed or unexpected data types, when the tool is run, then it should handle the discrepancy gracefully and document the column type appropriately.
