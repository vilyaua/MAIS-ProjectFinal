# User Story

> Build a REST API with Flask for a task manager with SQLite persistence, user authentication via API keys, and CRUD operations for tasks with priorities and due dates

## Specification: Task Manager REST API with Flask and SQLite

**Complexity:** medium

### Requirements

1. Implement a REST API using Flask framework to manage tasks.
2. Use SQLite as the database backend to persist task data and user information.
3. Implement user authentication using API keys; each request must include a valid API key for authorization.
4. Provide CRUD (Create, Read, Update, Delete) endpoints for tasks with fields: title, description, priority, and due date.
5. Validate input data for tasks, ensuring priority is within allowed range, due dates are valid dates, and required fields are present.
6. Handle errors gracefully, providing meaningful HTTP status codes and messages for invalid requests or authentication failures.
7. Ensure API supports listing tasks with optional filtering/sorting by priority and due date.

### Acceptance Criteria

1. Given a valid API key, when a task is created with valid data, then the API returns a success response with the created task's details.
2. Given an invalid or missing API key, when any API endpoint is accessed, then the API returns an unauthorized error response.
3. Given a request to update a task with invalid data (e.g., invalid date format), when the request is processed, then the API returns a validation error response.
4. Given a request to delete an existing task with a valid API key, when the request is processed, then the API returns a success response indicating task removal.
5. Given a request to retrieve a list of tasks with filtering parameters, when the request is processed, then the API returns a filtered list of tasks matching the criteria.
6. Given a request to read a specific task by ID with a valid API key, when the request is processed, then the API returns the task details if found or a not found error if the ID does not exist.
