# User Story

> Write a Python CLI tool that reads a CSV file and generates a statistical summary report: column types, min/max/mean/median, missing values, and top-5 most frequent values per column

## Specification: CSV Statistical Summary CLI Tool

**Complexity:** medium

### Requirements

1. The tool shall accept a CSV file path as a command-line argument.
2. The tool shall read and parse the CSV file correctly, handling different delimiters and quoted values.
3. The tool shall infer column data types (e.g., numeric, string, date) for each column.
4. The tool shall compute and report per column statistics: minimum, maximum, mean, median for numeric columns.
5. The tool shall identify and report the count or percentage of missing values for each column.
6. The tool shall identify and report the top-5 most frequent values for each column.
7. The tool shall handle errors gracefully, including file not found, parsing errors, and empty files.

### Acceptance Criteria

1. Given a valid CSV file with mixed data types, when the tool is run, then it produces a summary report containing column data types, statistics, missing values, and top-5 frequent values.
2. Given a CSV file with missing values, when the tool is run, then missing value counts are accurately reported per column.
3. Given a CSV file with numerical columns, when the tool is run, then min, max, mean, and median statistics are correctly calculated and displayed.
4. Given a CSV file with string columns, when the tool is run, then the top-5 most frequent values with counts are displayed for those columns.
5. Given a non-existent file path, when the tool is run, then a clear error message indicating the file is not found is shown.
6. Given a corrupted or improperly formatted CSV file, when the tool is run, then a parsing error message is displayed without crashing.
7. Given an empty CSV file, when the tool is run, then the tool reports that the file contains no data and exits gracefully.
