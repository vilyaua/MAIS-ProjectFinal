## Specification: Temperature Conversion Function

**Complexity:** simple

### Requirements

1. Implement a Python function that accepts three parameters: the temperature value (float or int), the current unit of the temperature (Celsius, Fahrenheit, or Kelvin), and the target unit to convert to (Celsius, Fahrenheit, or Kelvin).
2. The function must correctly convert temperatures between Celsius, Fahrenheit, and Kelvin based on standard conversion formulas.
3. Validate that the input temperature is a numeric value (int or float).
4. Validate that the source and target units are among the allowed units: 'C', 'F', 'K' (case-insensitive).
5. Handle invalid inputs by raising appropriate exceptions with clear error messages.
6. Include handling for edge cases, such as absolute zero and identical source and target units (which should return the original temperature).

### Acceptance Criteria

1. Given a temperature 0 Celsius, when converted to Fahrenheit, then the function returns 32.0.
2. Given a temperature 32 Fahrenheit, when converted to Celsius, then the function returns 0.0.
3. Given a temperature 273.15 Kelvin, when converted to Celsius, then the function returns 0.0.
4. Given a temperature value and the same source and target unit, when converted, then the function returns the original temperature without modification.
5. Given an invalid temperature input (e.g., a string not convertible to float), when the function is called, then it raises a ValueError with an appropriate message.
6. Given an invalid unit (e.g., 'X'), when the function is called, then it raises a ValueError with an appropriate message about invalid unit.
7. Given a temperature below absolute zero in any scale, when converting, then the function handles the input according to physical constraints (e.g., raise an exception or allow negative values only where physically possible).
