# User Story

> Напиши Python CLI-инструмент: на вход — Excel с платежами поставщику и спецификациями по контракту (схема 30% / 70%), на выходе — отчёт о расхождениях: недоплаты, переплаты, нулевые балансы при реальных долгах, неверно посчитанные предоплаты. Поддержка валют USD/CNY/EUR.

## Specification: Payment Reconciliation CLI Tool

**Complexity:** medium

### Requirements

1. Accept an Excel file as input containing supplier payments and contract specifications with a 30%/70% payment scheme.
2. Parse and validate payment data and contract specifications including payment amounts, currencies (USD, CNY, EUR), and payment percentages.
3. Calculate expected payments based on the contract scheme (30% initial, 70% final) and actual payments for each supplier.
4. Identify and report discrepancies: underpayments, overpayments, zero balances with outstanding debts, and incorrectly calculated prepayments.
5. Generate a summary report showing detail of all discrepancies per supplier and payment entry.
6. Handle input validation errors such as missing fields, unsupported currencies, and malformed Excel files gracefully, providing user-friendly error messages.
7. Support multiple currencies (USD, CNY, EUR) and handle currency data correctly during calculations.

### Acceptance Criteria

1. Given a valid Excel file with correct payment and contract data, when the tool is run, then it outputs a report listing no discrepancies if payments match expected amounts.
2. Given an Excel file containing underpayments, overpayments, zero balances with debts, or incorrect prepayments, when the tool processes the file, then those discrepancies are correctly identified and reported.
3. Given an Excel file with unsupported currency codes, when the tool is run, then it shows an informative error message and exits without crashing.
4. Given a malformed or corrupted Excel input file, when the tool attempts to parse it, then it gracefully handles the error and informs the user.
5. Given missing required columns or data in the Excel file, when the tool processes the input, then it reports the missing data clearly to the user.
6. Given payment data involving multiple currencies (USD, CNY, EUR), when calculating and reporting discrepancies, then currency values are properly handled and displayed in the report.
7. Given the contract specifies a 30%/70% payment scheme, when calculating expected payment milestones, then the tool computes and applies these percentages correctly for reconciliation.
