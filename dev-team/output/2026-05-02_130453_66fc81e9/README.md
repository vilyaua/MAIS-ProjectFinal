## Specification: Daily Expense Tracker CLI Tool

**Complexity:** medium

### Requirements

1. The CLI tool must allow users to input daily expenses with an amount and category.
2. The tool must support multiple expense categories and ensure category input validation.
3. The tool must generate monthly expense reports summarizing total and categorized expenses.
4. The tool must allow exporting the expense data for a specified period to a CSV file.
5. The tool must handle invalid inputs gracefully, including negative amounts and unsupported categories.
6. The tool must manage persistent storage of expenses to retain data across sessions.
7. The tool must include CLI help commands to guide users on usage.

### Acceptance Criteria

1. Given valid amount and category inputs, when the user adds an expense, then the expense is recorded and stored persistently.
2. Given a request for a monthly report, when the user specifies a month and year, then the tool generates a summary report including total and categorized expenses.
3. Given an export command, when the user specifies the period, then a CSV file is created with corresponding expenses.
4. Given invalid input such as negative amount or unsupported category, when the user tries to add an expense, then the tool shows an appropriate error message and does not record the expense.
5. Given the tool is invoked with --help or similar argument, when the user requests help, then the tool displays usage instructions and available commands.
