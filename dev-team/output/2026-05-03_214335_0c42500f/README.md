# User Story

> Write a Python CLI tool that reads a CSV file and generates a statistical summary report: column types, min/max/mean/median, missing values, and top-5 most frequent values per column

## Specification: CSV Statistical Summary CLI Tool

**Complexity:** medium

### Requirements

1. The CLI tool shall accept a path to a CSV file as an input argument.
2. The tool shall read and parse the CSV file correctly, handling common CSV formatting variations.
3. For each column in the CSV, the tool shall determine the data type (e.g., numeric, categorical, datetime).
4. The tool shall calculate and report the minimum, maximum, mean, and median for numeric columns.
5. The tool shall identify and report the number of missing values per column.
6. The tool shall identify and report the top-5 most frequent values and their counts for each column, regardless of type.
7. The tool shall handle errors gracefully, including file not found, unreadable files, and malformed CSV data.

### Acceptance Criteria

1. Given a valid CSV file with numeric and categorical data, when the tool is run, then it outputs the correct data types and descriptive statistics for each column.
2. Given a CSV file with missing values, when the tool is run, then it accurately reports the count of missing values per column.
3. Given a CSV file, when the tool is run, then it lists the top-5 most frequent values per column along with their frequencies.
4. Given an invalid file path, when the tool is run, then it returns a clear error message about the file not being found.
5. Given a malformed CSV file, when the tool is run, then it returns a relevant error message without crashing.
6. Given a CSV with columns of different types, when the tool is run, then it correctly identifies column types (numeric, categorical, datetime) with no misclassification.
