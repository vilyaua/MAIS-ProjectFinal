# User Story

> Create a Python library that converts Markdown text to HTML, supporting headers, bold, italic, links, code blocks, and lists

## Specification: Markdown to HTML Conversion Library

**Complexity:** medium

### Requirements

1. The library must accept a string input containing Markdown text.
2. It must support conversion of the following Markdown elements to HTML: headers (H1 to H6), bold text, italic text, hyperlinks, inline code, code blocks, and unordered and ordered lists.
3. The library should provide a function that returns the converted HTML as a string.
4. Input validation must ensure the input is a non-empty string; otherwise, an appropriate error should be raised.
5. The conversion should properly handle nested elements (e.g. bold text inside headers, links inside lists).
6. The library should handle malformed Markdown gracefully without crashing, either by ignoring invalid Markdown or raising descriptive errors.
7. The output HTML must be well-formed and valid for the supported Markdown elements.

### Acceptance Criteria

1. Given valid Markdown input with headers, when the conversion function is called, then it returns correctly formatted HTML headers (e.g. <h1>, <h2>, etc.).
2. Given Markdown input including bold and italic syntax, when converted, the output HTML uses <strong> for bold and <em> for italic text appropriately.
3. Given Markdown text with links, when converted, the output includes correct <a> tags with appropriate href attributes.
4. Given input with inline code and code blocks, when converted, inline code should be wrapped with <code> and code blocks within <pre><code> tags.
5. Given input containing ordered and unordered lists, when converted, the output contains <ol> and <ul> elements with nested <li> items.
6. Given an empty or non-string input, when the conversion function is called, then it raises a ValueError or similar exception indicating invalid input.
7. Given malformed Markdown input that does not conform to standard Markdown syntax, when converted, the library does not crash and outputs best-effort HTML or raises descriptive errors.
