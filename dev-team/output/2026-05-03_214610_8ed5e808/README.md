# User Story

> Build a REST API with Flask for a task manager with SQLite persistence, user authentication via API keys, and CRUD operations for tasks with priorities and due dates

## Specification: Task Manager REST API with Authentication and SQLite Persistence

**Complexity:** medium

### Requirements

1. Develop a REST API using Flask framework.
2. Implement user authentication using API keys for all endpoints.
3. Create SQLite database schema to store users, API keys, and tasks.
4. Support CRUD (Create, Read, Update, Delete) operations on tasks.
5. Each task should include attributes: title, description, priority, and due date.
6. Validate input data - ensure priority values and due dates are in acceptable formats.
7. Handle error cases such as invalid API key, missing fields, invalid data formats, and database errors gracefully with appropriate HTTP status codes.

### Acceptance Criteria

1. Given a valid API key, when a user sends a POST request with task details, then a new task is created and stored in SQLite.
2. Given an invalid or missing API key, when a user makes any API request, then the response returns 401 Unauthorized.
3. Given a valid API key, when a user sends GET request for tasks, then the API returns list of tasks associated with the user.
4. Given a valid API key, when a user sends PUT request with updated task attributes, then the task is updated correctly.
5. Given a valid API key, when a user sends DELETE request for a task id, then the task is deleted from database.
6. Given a request with invalid data (e.g., priority out of range or malformed date), when creating or updating a task, then the API returns 400 Bad Request with error description.
7. Given database errors during operations, when an API request is made, then the API responds with appropriate 5xx status and error message.
