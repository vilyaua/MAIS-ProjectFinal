# User Story

> Построй REST API на Django/Flask для управления цепочкой поставок: SKU, поставщики, контракты, контейнеры со статусами (произведено → отгружено → в пути → на таможне → получено), привязка к инвойсам и платежам. SQLite/PostgreSQL, API-key аутентификация, CRUD по всем сущностям, фильтры по статусам и датам, экспорт отчётов в Excel.

## Specification: Supply Chain Management REST API

**Complexity:** medium

### Requirements

1. Implement a REST API using Django or Flask framework.
2. Support CRUD operations for entities: SKU, suppliers, contracts, containers, invoices, and payments.
3. Containers should have status transitions: produced → shipped → in transit → in customs → received.
4. Use SQLite or PostgreSQL as the database backend.
5. Implement API-key based authentication for all endpoints.
6. Provide filtering functionality on container statuses and date fields in relevant entities.
7. Implement export of reports in Excel format capturing supply chain data.

### Acceptance Criteria

1. Given valid API keys, when sending CRUD requests for SKU, suppliers, contracts, containers, invoices, or payments, then the API responds with correct success/failure status and data changes.
2. Given container records, when filtered by status or date range, then the API returns only containers matching the filters.
3. Given requests without or with invalid API-keys, when accessing API endpoints, then the API returns an authentication error.
4. Given container status transitions, when updating container status, then only valid status sequences are accepted (no skipping backwards).
5. Given report export requests, when requesting export, then the API returns an Excel file containing relevant supply chain data.
6. Given invalid input data on any CRUD operation, when sending the request, then the API returns appropriate validation error messages.
