# User Story

> Build a CLI password generator that supports custom length, uppercase, lowercase, digits, and special characters with a strength meter

## Specification: CLI Password Generator with Strength Meter

**Complexity:** medium

### Requirements

1. Provide command-line options to specify password length and inclusion of uppercase letters, lowercase letters, digits, and special characters.
2. Validate input parameters: length must be a positive integer within a reasonable range (e.g., 8 to 128); at least one character type must be selected.
3. Generate a random password based on the given character set and length.
4. Implement a password strength meter that evaluates generated password based on length, variety of character types, and entropy criteria.
5. Display the generated password along with its strength classification (e.g., Weak, Moderate, Strong) in the CLI output.
6. Handle invalid inputs with descriptive error messages and usage hints.
7. Support edge cases such as the minimum and maximum length and when only one character type is selected.

### Acceptance Criteria

1. Given valid inputs for length and character types, when the password is generated, then the password matches the specified criteria in composition and length.
2. Given an invalid length input (e.g., negative number, zero, or above max), when generating the password, then the CLI outputs an error message and does not generate a password.
3. Given no character types selected, when attempting to generate a password, then the CLI outputs an error message indicating that at least one type must be chosen.
4. Given the generated password, when strength is evaluated, then the output classifies the password strength correctly based on predefined rules.
5. Given edge case inputs such as minimum length and only one character type, when generating the password, then it still produces a valid password and strength rating.
6. Given the user requests help or invalid options, when the CLI runs, then it displays usage instructions.
