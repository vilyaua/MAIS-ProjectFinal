# User Story

> Build a CLI password generator that supports custom length, uppercase, lowercase, digits, and special characters with a strength meter

## Specification: CLI Password Generator with Strength Meter

**Complexity:** medium

### Requirements

1. The CLI tool shall generate a password based on user-specified length.
2. The tool shall allow the user to include or exclude uppercase letters in the generated password.
3. The tool shall allow the user to include or exclude lowercase letters in the generated password.
4. The tool shall allow the user to include or exclude digits in the generated password.
5. The tool shall allow the user to include or exclude special characters in the generated password.
6. The tool shall evaluate and display a strength meter for the generated password based on length and character variety.
7. The tool shall validate user inputs and handle errors such as invalid length or lack of character types gracefully.

### Acceptance Criteria

1. Given valid inputs (length and character types), when the user requests a password, then a password of the specified length and character set is generated.
2. Given an input length less than the minimum allowed (e.g., less than 1), when the user inputs it, then the tool returns an error message and requests a valid length.
3. Given user choice that excludes all character types, when attempting to generate password, then the tool returns an error indicating at least one character type must be selected.
4. Given a generated password, when the password is evaluated, then the strength meter outputs a strength rating based on password complexity (e.g., weak, moderate, strong).
5. Given the CLI execution, when invalid inputs are entered, then the tool handles the error without crashing and prompts the user appropriately.
