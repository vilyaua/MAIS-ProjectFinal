# User Story

> ``` Create a Python library that converts Markdown text to HTML, supporting headers, bold, italic, links, code blocks, and lists ```

## Specification: Markdown to HTML Converter Library

**Complexity:** medium

### Requirements

1. Implement parsing of Markdown headers (#, ##, ###, etc.) and convert them to corresponding HTML header tags (<h1>, <h2>, etc.).
2. Support Markdown bold syntax (**bold** or __bold__) and convert it to <strong> HTML tags.
3. Support Markdown italic syntax (*italic* or _italic_) and convert it to <em> HTML tags.
4. Implement conversion of Markdown links [text](url) to HTML anchor <a> tags with href attributes.
5. Parse and convert fenced code blocks (```code```) into HTML <pre><code> blocks.
6. Handle unordered (starting with -, *, or +) and ordered lists (starting with numbers) and convert them to <ul>, <ol> and <li> tags accordingly.
7. Validate input to ensure it is a string and handle empty or null input gracefully by returning an appropriate empty HTML element or message.

### Acceptance Criteria

1. Given a Markdown string with headers, when passed to the library, then the output contains corresponding HTML header tags.
2. Given a Markdown string with bold and italic text, when passed to the library, then the output contains <strong> and <em> tags respectively.
3. Given a Markdown string with links, when passed to the library, then the output contains <a> tags with correct href attributes.
4. Given a Markdown string with fenced code blocks, when passed to the library, then the output wraps code within <pre><code> blocks.
5. Given a Markdown string with unordered and ordered lists, when passed to the library, then the output contains correctly nested <ul>, <ol>, and <li> tags.
6. Given null or empty input, when passed to the library, then the output is an empty string or valid empty HTML structure.
7. Given invalid non-string input, when passed to the library, then the library raises a TypeError with an appropriate error message.
