# User Story

> Build a CLI password generator that supports custom length, uppercase, lowercase, digits, and special characters with a strength meter

## Specification: CLI Password Generator with Strength Meter

**Complexity:** medium

### Requirements

1. The CLI tool must generate passwords with a user-defined length, supporting input validation to ensure length is a positive integer within a reasonable range (e.g., 8 to 128).
2. The password generation must allow inclusion of uppercase letters, lowercase letters, digits, and special characters, with user options to enable or disable each category.
3. The tool must calculate and display a password strength meter based on the selected criteria and generated password characteristics (e.g., length, variety of character types).
4. Input validation must handle invalid inputs gracefully, providing clear error messages and prompting for correction.
5. The tool should handle edge cases such as when no character types are selected by displaying an appropriate error and not generating a password.
6. The password generation must be random and secure, avoiding predictable patterns.
7. The CLI interface must be user-friendly, providing help or usage instructions when requested or when invalid parameters are supplied.

### Acceptance Criteria

1. Given valid inputs for length and character type options, when the user runs the CLI, then a password of the specified length and character composition is generated and displayed.
2. Given an input length outside the allowed range or non-integer, when the user attempts to generate a password, then the tool displays an error message and prompts for valid input.
3. Given no character types selected, when the user tries to generate a password, then the tool displays an error message indicating that at least one character type must be chosen.
4. Given a generated password, when the password is displayed, then a strength meter reflecting the password's estimated strength based on length and character variety is shown.
5. Given the user requests help or uses invalid command line options, when the CLI is run, then usage instructions are displayed explaining the options and how to use the tool.
6. Given repeated generation requests with the same parameters, when the CLI is run multiple times, then passwords generated should be different and random each time.
7. Given long password lengths (e.g., 128 characters), when the CLI generates a password, then the output should correctly handle and display the full generated password without truncation.
