# User Story

> Build a REST API with Flask for a task manager with SQLite persistence, user authentication via API keys, and CRUD operations for tasks with priorities and due dates

## Specification: Task Manager REST API with Flask and SQLite

**Complexity:** medium

### Requirements

1. Implement REST API endpoints using Flask for task management including create, read, update, and delete (CRUD) operations.
2. Persist task data in an SQLite database with fields for task description, priority, due date, and a unique identifier.
3. Implement user authentication via API keys, including verification of API keys on each request to authorized endpoints.
4. Validate input data for task creation and updates, ensuring priority is within a valid range and due dates are valid future dates.
5. Handle error cases gracefully, returning appropriate HTTP status codes and error messages for authentication failures, validation errors, and database errors.
6. Support filtering or sorting tasks by priority and due date in the retrieval endpoints.
7. Ensure thread-safe access to SQLite and secure storage of API keys.

### Acceptance Criteria

1. Given a valid API key and correct task data, when a create task request is made, then the API returns a success response with the created task details.
2. Given an invalid or missing API key, when any secured endpoint is accessed, then the API returns an authentication error with status 401 Unauthorized.
3. Given invalid task input data (e.g., negative priority, past due date), when a create or update request is made, then the API returns a validation error with status 400 Bad Request.
4. Given an existing task, when a read request for that task is made with a valid API key, then the API returns the correct task data with status 200 OK.
5. Given a request to update or delete a task with a valid API key, when the task exists, then the API updates or deletes the task and returns a success response.
6. Given a request to list tasks, when filtering or sorting parameters are provided, then the API returns tasks filtered or sorted by priority and due date as requested.
7. Given concurrent requests to the API, when accessing the SQLite database, then data integrity and consistency are maintained without errors.
