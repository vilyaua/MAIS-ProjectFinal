3. Given invalid characters or control sequences in input when the script runs then these are sanitized or removed before generating ASCII art.
4. Given a specified font name not available when the script runs then a fallback default font is used and a warning is shown.
5. Given normal text input when the script runs with default settings then ASCII art is printed using the default font and style.
6. Checklist: The script runs without crashing on any valid and invalid input strings.
7. Checklist: The output ASCII art is visually distinct and clearly represents the input text.
