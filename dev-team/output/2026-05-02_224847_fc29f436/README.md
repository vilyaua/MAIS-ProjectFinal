# User Story

> Write a Python CLI tool that reads a CSV file and generates a statistical summary report: column types, min/max/mean/median, missing values, and top-5 most frequent values per column

## Specification: CSV Statistical Summary CLI Tool

**Complexity:** medium

### Requirements

1. The tool shall accept a CSV file path as input via the command line interface.
2. The tool shall read and parse the CSV file correctly, handling different data types per column.
3. The tool shall identify and report the data type of each column (e.g., numeric, categorical).
4. The tool shall calculate and report the minimum, maximum, mean, and median values for numeric columns.
5. The tool shall identify and report the number of missing/null values per column.
6. The tool shall identify and report the top-5 most frequent values for each column, regardless of type.
7. The tool shall handle errors gracefully for invalid file paths, non-CSV files, or malformed CSV input, providing user-friendly error messages.

### Acceptance Criteria

1. Given a valid CSV file with numeric and categorical data, when the tool is run with the file path, then it outputs column types, min/max/mean/median for numeric columns, missing values count for all columns, and top-5 frequent values for each column.
2. Given a CSV file containing missing/null values, when the tool processes the file, then it accurately reports the count of missing values per column.
3. Given a CSV file with non-numeric columns, when the tool is run, then it reports column types as categorical and lists the top-5 most frequent values.
4. Given a non-existent file path, when the tool is run, then it returns a clear error message indicating the file could not be found.
5. Given a malformed or corrupted CSV file, when the tool attempts to read it, then it returns an error message indicating the file could not be parsed.
6. Given an empty CSV file, when the tool is run, then it handles gracefully by indicating no data to process rather than crashing.
