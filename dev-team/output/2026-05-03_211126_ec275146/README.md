## Specification: Invoice and Packing List Reconciliation Library for Chinese Suppliers

**Complexity:** complex

### Requirements

1. Develop a Python library capable of parsing commercial invoices and packing lists from Excel and PDF files provided by Chinese suppliers.
2. The library must parse relevant fields including item codes (артикулы), quantities, and prices from the documents.
3. Implement comparison functionality that matches parsed items against an original order data structure containing expected item codes, quantities, and prices.
4. Return a structured list of discrepancies highlighting mismatches in item codes, quantities, or prices.
5. Validate input file formats and handle errors gracefully, providing meaningful error messages for unsupported file types or parsing failures.
6. Handle edge cases such as missing or extra items in the invoice or packing list compared to the original order.
7. Ensure that numeric fields such as quantities and prices are correctly parsed and validated, including proper handling of different decimal and thousand separators common in Chinese documents.

### Acceptance Criteria

1. Given an Excel invoice file and an original order, when the library parses and compares them, then it returns a list of discrepancies where item codes, quantities, or prices differ.
2. Given a PDF packing list and a matching original order, when parsing is successful, then all matching items with correct quantities and prices yield no discrepancies in the returned list.
3. Given input files in unsupported formats, when processed by the library, then it raises a clear error indicating unsupported file types.
4. Given an invoice containing items missing from the original order, when compared, then those items are flagged as discrepancies indicating missing expected items.
5. Given numeric fields with different formatting (e.g., comma as decimal separator), when parsed, then quantities and prices are accurately read without error.
6. Given corrupted or unreadable Excel or PDF files, when parsing is attempted, then the library returns a meaningful error rather than crashing.
